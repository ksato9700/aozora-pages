# Design: sjis_to_utf8 Improvements

**Status:** Proposed
**Date:** 2026-03-15
**Context:** Analysis of `aozora_data/sjis_to_utf8/` prior to planned rework.

---

## 1. Dead code: `load_gaiji_table()` and its call site in `get_gaiji()`

**Severity:** Low — no functional impact, but misleading.

**Location:** `converter.py`, lines 13–25 and 37–38

`load_gaiji_table()` was originally responsible for reading a `jisx0213-2004-std.txt` file at runtime. The table is now pre-compiled into `gaiji_table.py` and imported as a module-level constant. The function body is effectively a no-op when the import succeeds (which it always does), and the fallback path (`GAIJI_TABLE = {}`) silently disables gaiji resolution with no warning.

Similarly, the guard `if not GAIJI_TABLE: load_gaiji_table()` inside `get_gaiji()` is never triggered.

**Fix:** Remove `load_gaiji_table()` entirely. Remove the `try/except ImportError` wrapper around the `gaiji_table` import and let an import failure raise loudly. Remove the `if not GAIJI_TABLE` guard in `get_gaiji()`.

---

## 2. Bug: comment labels "Pattern 2" twice in `get_gaiji()`

**Severity:** Low — cosmetic, no functional impact.

**Location:** `converter.py`, lines 70 and 90

The function has three resolution patterns, but the comments label them Pattern 1, Pattern 2, and Pattern 2 again:

```python
# Pattern 1: JIS X 0213 plane/row/cell (第N水準)   ← line 46
# Pattern 2: Generic JIS X 0213 plane/row/cell       ← line 70
# Pattern 2: Direct Unicode Reference                 ← line 90  ← wrong
```

**Fix:** Renumber the third block to "Pattern 3: Direct Unicode Reference".

## 3. `sub_gaiji()` is O(n²) for documents with many annotations

**Severity:** Medium — noticeable for texts with hundreds of gaiji annotations.

**Location:** `converter.py`, lines 100–168

The outer `while True` loop re-scans the entire string from the start on every iteration. In the common case (no nesting) a single pass suffices, but the loop always runs at least twice (once making substitutions, once confirming no further matches). For a document with N annotations this is O(N) iterations × O(N) scan = O(N²).

**Fix:** Replace the iterative loop with a single-pass `re.sub()` using a callable replacement function:

```python
def sub_gaiji(text: str) -> str:
    pattern = re.compile(r"(?:※)?［＃(?:(?!［＃).)+?］")

    def replace(m: re.Match) -> str:
        annotation = m.group(0)
        return get_gaiji(annotation)

    return pattern.sub(replace, text)
```

Placeholder removal (stripping the preceding character) can be handled inside `replace()` by returning a zero-width result and using a lookbehind, or by keeping a second targeted pass only for the placeholder pattern. Since placeholder annotations are rare, a second targeted pass is simpler and still O(N).

Note: nested annotations are extremely rare in practice. If support for them is still required, the single-pass approach can be wrapped in a loop that only runs additional passes when the previous pass made at least one change — but this is unlikely to be needed.

## 4. Missing `__main__.py` — `python -m aozora_data.sjis_to_utf8` does not work

**Severity:** Low — `uv run sjis-to-utf8` works via the console script entry point, but the module invocation is not supported (unlike `text_to_html`, which has a `__main__.py` added in a prior fix).

**Fix:** Add `aozora_data/sjis_to_utf8/__main__.py`:

```python
from .cli import main
main()
```

## 5. `convert_all.py` skips by existence, not by modification time

**Severity:** Low — inconsistent with `html_convert_all.py`, which skips only when the output is newer than the input.

**Location:** `aozora_data/convert_all.py`, line 43

```python
if output_path.exists():
    count_skipped += 1
    continue
```

If a source SJIS file is updated (e.g., a corrected Aozora Bunko release), the existing UTF-8 output will never be refreshed.

**Fix:** Mirror `html_convert_all.py`'s mtime check:

```python
if output_path.exists():
    if sjis_path.stat().st_mtime <= output_path.stat().st_mtime:
        count_skipped += 1
        continue
```

---

## 6. `generate_gaiji_table.py` is referenced but absent from the repo

**Severity:** Medium — the header of `gaiji_table.py` says it is auto-generated by `scripts/generate_gaiji_table.py`, but that script does not exist in the repository. Without it, the table cannot be regenerated if the upstream JIS X 0213 mapping file changes.

**Fix:** Either add the generator script to `scripts/`, or document the generation steps in `docs/SJIS_TO_UTF8.md` so the table can be reproduced.

---

## 7. Fullwidth backslash placeholder is fragile

**Severity:** Low — extremely unlikely to cause issues in practice, but worth noting.

**Location:** `converter.py`, line 183

```python
placeholder = b"_AA_FWBS_AA_"
```

The placeholder is a fixed ASCII string substituted into the byte stream before decoding. If a Shift JIS source file legitimately contained this exact byte sequence (as ASCII text), it would be incorrectly replaced with `＼` after decoding.

**Fix:** Use a placeholder that cannot appear in valid Shift JIS input — for example, a null byte (`\x00`) or a sequence beginning with a null byte is not valid in Shift JIS text files and would serve as a safe sentinel.

## Implementation Plan

### Commit 1 — Housekeeping (issues 1, 2, 4, 5, 7) ✓ Implemented

All small, safe changes with no algorithmic impact.

**Issue 1 — Remove dead code**

Delete `load_gaiji_table()`. Replace the `try/except ImportError` import with a plain import (let it raise loudly on failure). Remove the `if not GAIJI_TABLE: load_gaiji_table()` guard in `get_gaiji()`.

**Issue 2 — Fix comment numbering**

Rename the third block comment in `get_gaiji()` from "Pattern 2" to "Pattern 3".

**Issue 4 — Add `__main__.py`**

Add `aozora_data/sjis_to_utf8/__main__.py` with two lines:

```python
from .cli import main
main()
```

**Issue 5 — Fix mtime check in `convert_all.py`**

Replace the existence-only skip with the same mtime guard used in `html_convert_all.py`:

```python
if output_path.exists():
    if sjis_path.stat().st_mtime <= output_path.stat().st_mtime:
        count_skipped += 1
        continue
```

**Issue 7 — Safer fullwidth backslash sentinel**

Replace the ASCII string placeholder `b"_AA_FWBS_AA_"` with a single null byte `b"\x00"`. Null bytes do not appear in valid Shift JIS text files, making collision impossible. Update the restore step accordingly:

```python
placeholder = b"\x00"
# ...
text = text.replace("\x00", "＼")
```

---

### Commit 2 — Single-pass `sub_gaiji()` (issue 3) ✓ Implemented

Replace the O(n²) `while True` loop with a two-step single-pass approach:

**Step 1 — Resolve gaiji annotations in one pass**

Use `re.sub()` with `get_gaiji` as the callable replacement. This eliminates the outer loop for the common case (no nesting, no placeholder):

```python
pattern = re.compile(r"(?:※)?［＃(?:(?!［＃).)+?］")
text = pattern.sub(lambda m: get_gaiji(m.group(0)), text)
```

**Step 2 — Remove placeholders in a second targeted pass**

Placeholder removal requires erasing text *before* the annotation match, which `re.sub()` cannot do in a single replacement. Handle this with a second regex that captures both the placeholder and the annotation together:

```python
# Matches: <placeholder>［＃「<placeholder>」は..., resolved to short char］
placeholder_pattern = re.compile(r"(.{1,10})(?=［＃「\1」[はの])")
```

Iterate over matches, and for each one where the annotation resolves to a short substitution, remove the captured placeholder from the output.

Because placeholder annotations are rare and typically short, this second pass remains O(N) overall.

Nesting support (extremely rare in practice) can be preserved by wrapping Step 1 in a loop that only runs additional iterations if a substitution was made in the previous pass and nested annotations are detected.

---

### Commit 3 — Restore `generate_gaiji_table.py` (issue 6) ✓ Implemented

Add `scripts/generate_gaiji_table.py` to the repository. The script should:

1. Accept the path to the upstream `jisx0213-2004-std.txt` mapping file as an argument
2. Parse its lines (tab-separated: `key\tcodepoints\t...`)
3. Resolve multi-codepoint entries and surrogate pairs to Python Unicode escape sequences
4. Write `aozora_data/sjis_to_utf8/gaiji_table.py` with the auto-generated header and `GAIJI_TABLE = {...}`

Document the source of `jisx0213-2004-std.txt` (it is distributed with the `nkf` project and other JIS tools) in a comment at the top of the generator script.
