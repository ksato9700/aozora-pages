# Aozora Pages вЂ” Project Overview

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Astro 6.0 (static site generation) |
| Language | TypeScript 5 / Python 3.13 |
| Data source | JSON files (generated by import pipeline) |
| Search | Algolia (client-side, full-text search across books and persons) |
| Hosting | Cloudflare Pages (CDN edge, static files) |

| HTML mirror | `aozora.ksato9700.com` (UTF-8 HTML conversion) |
| Import pipeline | Python (uv), runs as a Cloud Run Job |
| CI/CD | Google Cloud Build |

---

## Application Routes

| Route | Description |
|---|---|
| `/` | Home page вЂ” 24 most-recently added books + search bar |
| `/books/new/[page]` | Paginated list of recently added books |
| `/books/[bookId]` | Book detail вЂ” metadata, contributors, download links |
| `/persons/[personId]` | Author profile вЂ” bio and works grouped by role |
| `/read/[bookId]` | In-browser reader вЂ” HTML mode (iframe) or text mode (vertical) |

All pages are pre-rendered at build time. The reader (`/read/*`) is a client-side SPA served via a single HTML shell + Cloudflare Pages `_redirects` rewrite.

---

## Key Features

### Search
`SearchSection.tsx` (React Island) debounces user input and calls Algolia directly from the browser using the public Search-Only API key. Results appear in a live dropdown. No server is involved.

### Reader
`/read/[bookId]` supports two modes selected via `?format=` query param:

- **HTML mode** (default): embeds an `<iframe>` pointing to `aozora.ksato9700.com` (UTF-8 HTML mirror).
- **Text mode**: fetches the `.txt` or `.zip` file via `/api/read` (a Cloudflare Pages Function), decodes Shift_JIS, and renders the content vertically.

### Content Delivery
For books without copyright restrictions (`book.copyright === false`), the app constructs the URL from the book ID: `https://aozora.ksato9700.com/{bookId}.utf8.html`.

## Data Model (JSON Files)

Three JSON files are generated by the import pipeline and read by Astro at build time:

- **`books.json`** вЂ” array of all books; keyed by `book_id` (zero-padded string). Includes denormalized `author_name` / `author_id`.
- **`persons.json`** вЂ” array of all persons; keyed by `person_id` (zero-padded string).
- **`contributors.json`** вЂ” array of bookвЂ“personвЂ“role relationships.

See [DATA_FORMAT.md](./DATA_FORMAT.md) for the full field reference.

---

## Search Index (Algolia)

Two Algolia indices mirror a subset of the JSON data for full-text search:

- **`books`** вЂ” `title`, `title_yomi`, `author_name`
- **`persons`** вЂ” `last_name`, `first_name`, `last_name_yomi`, `first_name_yomi`

Indices are updated incrementally after each daily import run (only changed records are sent).

---

## Daily Pipeline

The full pipeline runs once a day as a **Cloud Run Job**, triggered by Cloud Scheduler:

```
1. Python: download CSV from aozora.gr.jp
2. Python: read watermark.json from R2 (last processed date)
3. Python: parse CSV, upsert all records to AozoraJSON (in memory)
4. Python: send only changed records to Algolia
5. Python: flush JSON files to /astro/data/
6. Python: write new watermark.json to R2
7. Node.js: npx astro build  в†’  generates ~19,000 HTML files in dist/
8. Node.js: wrangler pages deploy dist/ --project-name=aozora-pages
```

See [SETUP_CLOUDRUN.md](./SETUP_CLOUDRUN.md) for infrastructure setup.
